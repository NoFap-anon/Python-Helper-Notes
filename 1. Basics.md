# Базовые элементы и понятия

## 1. Ввод – вывод данных

### 1.1. Функция print()

Вывод данных в Питоне осуществляется куда проще чем в большинстве прочих языков программирования. Для сравнения можете зайти на эут страничку – http://helloworldcollection.de/

Вывод на экран осуществляет встроенная функция (built-in function) **print()**.

**Input:**

```Python
print("Hello World")
```
**Output:**
```Python
Hello World
```

Для вывода текста на экран этот текст сначала нужно окружить кавычками. Можно использовать как двойные "text", так и одинарные 'text' кавычки, но не смешивать их с друг другом.

То, что записано внутри **print()** называется **аргументом функции**.

***

Можно отправить на вывод сразу несколько аргументов внутри функции print():

**Input:**
```Python
print("What a time", "to be", "alive!")
print("What a time" + " " + "to be" + " " + "alive!")
print("What a time " + "to be " + "alive!")
```
**Output:**
```Python
What a time to be alive!
What a time to be alive!
What a time to be alive!
```

В первом случае аргументы внутри print() перед выводом соединяются через запятую. В этом случае при выводе аргументы автоматически разделяются пробелами.

Во втором случае соединение идет с помощью плюсов и пустого пробела внутри кавычек – + " " + .

В третьем случае соединение осуществляеся тоже при помощи плюсов, но плюсы включены в сами аргументы-строки.

Судя по моему скромному опыту, наиболее предпочтительным является второй способ.

***

**Если внутри текста нужно вывести одинарные или двойные кавычки**, то сам текст внутри print() нужно окружить другими кавычками, отличными от тех, что используются внутри самого текста:

**Input:**
```Python
print('Это "двойные" кавычки.')
print("А это – 'одинарные'.")
```
**Output:**
```Python
Это "двойные" кавычки.
А это – 'одинарные'.
```

***

**sep и end**

Внутри функции print() иногда бывает полезно использовать 2 необязательных параметра под названием **sep** (от англ. separator) и **end** (end of line).

**Input:**
```Python
print('a', 'b', 'c')
print('a', 'b', 'c', sep='/|\')
print('a', 'b', 'c', sep='---')
```
**Output:**
```Python
a b c
a|||b|||c
a---b---c
```

sep будет работать только если аргументы внутри  print() соединены при помощи запятых, а не плюсов.

По умолчанию sep равен пустому пробелу – **sep=' '**. Т.е. sep со значением по умолчанию как раз и осуществляет соединение аргументов функции print() добавляя между ними пробел. Но, если нам нужно соединить между собой аргументы как-то иначе (например, через "-" при обработке телефонных номеров), то sep можно указать в явной форме с заданным нами значением.

end по умолчанию имеет следующее значение – **end='\n'**. Т.е. end со значением по умолчанию отвечает за переход вывода на новую строку после завершения выполнения одной отдельной функции print(). Но если нам это не нужно, то можно поменять его значение на произвольное:

**Input:**
```Python
print('Hey, dude!', end=' First_stop ')
print('Where is the new line, dude?!', end=' Second_stop ')
print('WTF, dude?!!', end=' Third_stop ')
```
**Output:**
```Python
Hey, dude! First_stop Where is the new line, dude?! Second_stop WTF, dude?!! Third_stop
```

Для создания сразу нескольких новых строк после print() можно использовать end со слеюущими значениями – **end='\n\n\n'**.


Примечания:

1. В записи print("What a time", "to be", "alive!") пробелы после запятых не выводятся на экран. Их лучше ставить по соглашению PEP 8 для повышения читабельности кода.
2. Функцию print() обязательно нужно записывать только с маленькой буквы, т.к. Python чувствителен к регистру при вызове тех или иных функций.
3. Каждый новый print() выводит содержимое на новой строке.
4. Пустой print() выводит пустую строку (бывает полезно для визуального разделения блоков информации при выводе в консоль).
